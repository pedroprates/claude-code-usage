CC Usage Tracker
A native macOS menu bar app

A lightweight SwiftUI menu-bar utility that tracks Claude Code's 5-hour and weekly usage limits at a glance, and surfaces your OpenRouter balance on click. Sits in the menu bar, reads local ~/.claude logs, and respects the macOS Liquid Glass aesthetic.

Swift / SwiftUI macOS 14+ (Sonoma) MenuBarExtra Local JSONL parsing Keychain-stored API key No telemetry

01The menu bar item

A single compact status item — just a percentage and a colored dot. The percentage is the 5-hour window (white when low, yellow past 75%, red past 85%). The dot reflects the weekly limit's health. Everything else is one click away.

03Architecture

Three small services feed one SwiftUI view. No external dependencies, no backend, no network calls except the optional OpenRouter /credits fetch.

ClaudeUsageService CORE

  • Scans ~/.claude/projects/**/*.jsonl
  • Parses each line's timestamp + usage.input/output_tokens
  • Computes 5-hour sliding window token sum + % of cap
  • Computes rolling 7-day token sum + % of cap
  • Buckets per-hour / per-day for the sparklines
  • Re-scans on a 30s timer + FSEvents on file change

OpenRouterService OPTIONAL

  • Key stored in Keychain (account: cc-usage.openrouter)
  • GET https://openrouter.ai/api/v1/credits
  • Parses total_credits / total_usage
  • Refresh on click + every 5 min while panel open
  • Fails silently if no key configured

SettingsStore PERSIST

  • @AppStorage in UserDefaults
  • Plan type → limit presets (Pro / Max / custom)
  • Custom token caps override
  • Refresh interval, notification thresholds
  • Launch-at-login toggle

MenuBarView UI

  • MenuBarExtra with .window style for vibrancy
  • Compact label: "\(pct)%" + colored Circle
  • Popover: progress bars + Charts framework sparklines
  • Settings sheet for plan + OpenRouter key
  • UserNotifications on threshold cross (opt-in)
